# Factory Droid Session Manager 项目介绍

## 项目概述

Factory Droid Session Manager 是一个面向 `Factory Droid` 会话数据的可视化管理工具，提供网页端与桌面端两种使用方式。它用于扫描本地会话目录、展示会话列表、预览消息内容、查看统计分析，并支持删除空会话或单条会话。

## 解决的问题

当本地积累了大量 Droid 会话后，直接在文件系统中查找和管理会比较低效。这个项目把原始的会话文件整理成可浏览、可筛选、可分析的界面，方便快速定位重要会话和清理无效数据。

## 核心功能

- 自动扫描本地 `sessions` 目录
- 会话卡片列表展示
- 会话消息预览与详情查看
- 基于模型、消息数、Token、体积、工作区的统计分析
- 工作区总 token 用量排行，并分开展示输入 / 输出 / 缓存
- 支持删除单个会话
- 支持一键删除空会话
- 支持自动刷新、刷新间隔自定义
- 支持在会话详情中查看并复制完整 CLI 恢复命令
- 顶部全局总览与可视化图表
- 支持桌面端封装运行
- 支持 macOS / Windows / Linux 本地原生打包

## 技术实现

### 后端

- 基于 Python 标准库 `http.server`
- 默认网页模式不依赖第三方 Python 包
- 提供本地 API：
  - `/api/sessions`
  - `/api/session`
  - `/api/cli_command`
  - `/api/open_cli`
  - `/api/delete`
  - `/api/delete_empty`
- 提供前端静态资源：
  - `/`
  - `/index.html`
  - `/assets/...`

### 前端

- `frontend/index.html`
- `frontend/assets/style.css`
- `frontend/assets/app.js`
- 使用独立 HTML / CSS / JavaScript 页面
- 提供会话列表、筛选、统计卡片和图表分析

### 桌面端

- 使用 `pywebview`
- 由 `session-manager-desktop.py` 启动内嵌本地服务并创建原生窗口
- 支持 `--self-test` 与 `--sessions-dir`

### 打包

- 使用 `PyInstaller`
- `build-desktop.py` 统一处理依赖安装、自检、打包与归档输出
- `build-local-macos.sh` / `build-local-linux.sh` / `build-local-windows.bat` 都会转发到统一构建入口

## 跨平台设计

项目已经针对跨平台使用做了处理：

- Windows / macOS / Linux 使用不同的本地打包产物
- 桌面端支持在目标系统本机原生构建
- 默认端口启动时可自动结束旧的本项目进程
- `sessions` 目录不再写死，支持自动发现和手动覆盖

可通过以下方式指定自定义会话目录：

- 环境变量：
  - `FSM_SESSIONS_DIR`
  - `DROID_SESSIONS_DIR`
  - `FACTORY_SESSIONS_DIR`
- 启动参数：
  - `--sessions-dir`

可通过以下环境变量覆盖 Droid CLI 路径：

- `FSM_DROID_BIN`
- `DROID_BIN`
- `FACTORY_DROID_BIN`

## 主要文件说明

- `session-manager.py`：后端服务与本地 API
- `frontend/index.html`：前端页面
- `frontend/assets/style.css`：前端样式
- `frontend/assets/app.js`：前端脚本
- `session-manager-desktop.py`：桌面端入口
- `build-desktop.py`：桌面端打包脚本
- `build-local-macos.sh`：macOS 本地构建脚本
- `build-local-linux.sh`：Linux 本地构建脚本
- `build-local-windows.bat`：Windows 本地构建脚本
- `launch-session-manager-desktop.sh` / `.bat`：桌面端启动脚本
- `restart-session-manager.sh`：网页端服务重启脚本
- `README.md`：本地多机原生构建说明

## 依赖说明

- 网页端：仅 Python 标准库
- 桌面端运行依赖：`requirements-desktop.txt`
- 桌面端构建依赖：`requirements-build.txt`

## 适用场景

- 本地管理大量 Droid 会话
- 按项目或工作区查找历史上下文
- 分析不同模型的使用情况
- 清理无消息或无价值的会话数据
- 将工具打包为桌面应用分发给其他用户

## 项目特点

- 轻量，依赖少
- 本地运行，数据不出本机
- 同时支持网页端和桌面端
- 支持多平台本地构建
- 对自定义 `sessions` 路径更友好

## 总结

这是一个围绕 Factory Droid 会话数据管理而构建的本地工具项目，目标是让会话浏览、分析、清理和桌面化使用更加直观、高效、跨平台。
